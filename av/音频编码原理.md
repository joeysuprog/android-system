## 1. 概述与架构全景

### 1.1 什么是音频编码

音频编码是将声音信号（模拟信号）转换为数字信号，并进行压缩存储的过程。目的是在保证可接受音质的前提下，大幅减少数据量，以满足存储和传输的需求。

## 2. 声音基础

### 2.1 声音的本质

声音是由物体振动而产生的，是一种压力波。物体的振动会引起空气有节奏的振动，使周围的空气产生疏密变化，形成疏密相间的纵波。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    声音的产生与传播                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  振动源                    空气分子                    人耳         │
│                                                                      │
│    ┌─┐                                                   ┌───┐      │
│    │ │ ←→               疏 密 疏 密 疏 密               │   │      │
│    │ │振动   ─────→    ○○●●○○●●○○●●   ─────→   │ 鼓 │      │
│    │ │                                                   │ 膜 │      │
│    └─┘                                                   └───┘      │
│   音叉                     压力波                       听觉感知    │
│                                                                      │
│  声波 = 纵波（振动方向与传播方向相同）                             │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │   压力                                                       │   │

│  │  ──┼────╱─────╲──╱─────╲──╱─────╲────────→ 时间           │   │
│  │    │  ╱       ╲╱       ╲╱       ╲                          │   │
│  │    │ ╰─╯      ╰─╯      ╰─╯                                 │   │
│  │    ↓                                                        │   │
```

## 3. 音频数字化三步骤

### 3.1 采样（Sampling）

采样是在时间轴上对信号进行数字化，将连续的模拟信号按固定时间间隔取值，转换为离散信号。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    采样原理                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  模拟信号                                                           │
│  ────────                                                           │
│                         ╭──╮                                        │
│                        ╱    ╲     ╭─╮                               │
│  连续波形 ──→        ╱      ╲   ╱   ╲                              │
│                     ╱        ╲ ╱     ╲  ╱                           │
│  ───────────────────╱──────────╳───────╲╱────────────               │
│                                                                      │
│                          │ 采样（每隔固定时间取一个值）             │
│                          ↓                                          │
│                                                                      │
│  离散信号（采样后）                                                 │
│  ──────────────────                                                 │
│                         █                                           │
│                        █ █     █                                    │
│  样本值 ──→          █   █   █ █                                   │
│                     █     █ █   █                                   │
│  ─────────────────█───────────█────────────                         │
│                   ↑     ↑     ↑     ↑     ↑                        │
│                   T    2T    3T    4T    5T   (采样间隔 T)          │
│                                                                      │
│  采样率 = 1/T = 每秒采样次数                                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

#### 3.1.1 采样率（Sample Rate）

```
┌─────────────────────────────────────────────────────────────────────┐
│                    采样率与奈奎斯特定理                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  奈奎斯特采样定理（Nyquist Theorem）：                              │
│  ───────────────────────────────────────                            │
│  采样频率 ≥ 2 × 信号最高频率，才能无失真地还原原始信号            │
│                                                                      │
│  人耳听觉范围：20Hz ~ 20kHz                                         │
│  理论采样率 ≥ 2 × 20kHz = 40kHz                                    │
│  实际采样率 = 44.1kHz（留有余量）                                   │
│                                                                      │
│  常见采样率：                                                       │
│  ─────────────                                                      │
│  ┌─────────────┬────────────────────────────────────────────┐      │
│  │  采样率     │  应用场景                                   │      │
│  ├─────────────┼────────────────────────────────────────────┤      │
│  │  8 kHz      │  电话语音、语音通话                        │      │
│  │  16 kHz     │  VoIP、语音识别                            │      │
│  │  22.05 kHz  │  低质量音乐                                │      │
│  │  44.1 kHz   │  CD 音质（标准）                           │      │
│  │  48 kHz     │  DVD、专业音频、视频配音                   │      │
│  │  96 kHz     │  高清音频、专业录音                        │      │
│  │  192 kHz    │  发烧级音频                                │      │
│  └─────────────┴────────────────────────────────────────────┘      │
│                                                                      │
│  为什么 CD 采样率是 44.1kHz？                                       │
│  ─────────────────────────────                                      │
│  • 人耳最高听觉频率 20kHz × 2 = 40kHz（最低要求）                  │
│  • 44.1kHz 提供 10% 的保护带宽                                     │
│  • 历史原因：早期用视频设备存储音频，44.1kHz 与视频帧率兼容        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

#### 3.2.1 位深度（Bit Depth）

```
┌─────────────────────────────────────────────────────────────────────┐
│                    位深度与量化精度                                  │

│                                                                      │

│                                                                      │
│  ┌─────────────┬───────────────┬────────────────────────────┐      │

│                                                                      │

│    ┌───┐                           ┌─┐                              │
│    │   │                           ├─┤                              │

│    │   │                           ├─┤                              │

│  ─────────────────                                                  │
```

## 5. 音频压缩原理

### 5.1 压缩基本概念

```
┌─────────────────────────────────────────────────────────────────────┐
│                    压缩基本概念                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  压缩比 = 压缩后大小 / 原始大小                                     │
│  通常 < 1（如 0.1 表示压缩到原来的 1/10）                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  有损压缩 (Lossy Compression)                               │   │
│  │  ─────────────────────────────                              │   │
│  │  • 删除不重要的数据来压缩                                   │   │
│  │  • 解压后无法完全恢复原始数据                               │   │
│  │  • 压缩比高（可达 10:1 ~ 20:1）                            │   │
│  │  • 例子：MP3, AAC, OPUS                                    │   │
│  │                                                              │   │
│  │  类比：删除文章中的助词，意思大致保留但无法完全还原         │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  无损压缩 (Lossless Compression)                            │   │
│  │  ──────────────────────────────                             │   │
│  │  • 优化数据排列方式来压缩                                   │   │
│  │  • 解压后可完全恢复原始 PCM 数据                           │   │
│  │  • 压缩比较低（约 2:1 ~ 3:1）                              │   │
│  │  • 例子：FLAC, ALAC, APE                                   │   │
│  │                                                              │   │
│  │  类比：改变文章排版，内容完全不变                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  压缩编码的本质 = 去除冗余信息                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```
│                                                                      │
│  5. 长时自相关性                                                    │
│  ─────────────────                                                  │
│                                                                      │
│     在较长时间间隔（几十秒）内统计                                 │
│     相邻样值平均相关系数可达 0.9                                   │
│                                                                      │
```

```
│     • 前掩蔽：强音之前约 20ms 内的弱信号被掩蔽                    │

│     • 后掩蔽：强音消失后约 100-200ms 内的弱信号仍被掩蔽           │
│                                                                      │
│     被掩蔽的弱信号即为冗余信息                                     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 6. PCM 编码详解

### 6.1 PCM 原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PCM 编码原理                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  PCM (Pulse Code Modulation) = 脉冲编码调制                         │
│                                                                      │
│  PCM 是数字音频的基础格式，也被称为"无损编码格式"                  │
│  注意：PCM 并非绝对无损，只是最大程度接近原始声音                   │
│                                                                      │
│  三步过程：                                                         │
│  ──────────                                                         │
│                                                                      │
│     ┌───────────────────────────────────────────────────────────┐  │
│     │                                                            │  │
│     │  模拟信号                                                  │  │
│     │      │                                                     │  │
│     │      ↓ ① 采样 (Sampling)                                   │  │
│     │      │   以固定时间间隔取样                                │  │
│     │      │   44100 次/秒                                       │  │
│     │      ↓                                                     │  │
│     │  离散信号                                                  │  │
│     │      │                                                     │  │
│     │      ↓ ② 量化 (Quantization)                               │  │
│     │      │   将幅度映射到离散等级                              │  │
│     │      │   16bit = 65536 级                                  │  │
│     │      ↓                                                     │  │
│     │  量化信号                                                  │  │
│     │      │                                                     │  │
│     │      ↓ ③ 编码 (Encoding)                                   │  │
│     │      │   以二进制格式存储                                  │  │
│     │      │   顺序存储每个采样值                                │  │
│     │      ↓                                                     │  │
│     │  PCM 数据                                                  │  │
│     │                                                            │  │
│     └───────────────────────────────────────────────────────────┘  │
│                                                                      │
│  PCM 特点：                                                         │
│  ──────────                                                         │
│  • 最高保真水平的音频格式                                          │
│  • 广泛用于素材保存和音乐制作                                      │
│  • 数据量大，需要进一步压缩才适合传输                              │
│  • 是其他编码格式的基础                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.2 PCM 数据格式

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PCM 数据存储格式                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 交错格式 (Interleaved / Packed)                                 │
│  ──────────────────────────────────                                 │
│                                                                      │
│     不同声道的数据交错排列                                         │
│                                                                      │
│     ┌────┬────┬────┬────┬────┬────┬────┬────┬────────┐            │
│     │ L0 │ R0 │ L1 │ R1 │ L2 │ R2 │ L3 │ R3 │ ...    │            │
│     └────┴────┴────┴────┴────┴────┴────┴────┴────────┘            │
│                                                                      │
│     L = 左声道采样，R = 右声道采样                                 │
│                                                                      │
│     特点：                                                          │
│     • 每个采样时刻的所有声道数据连续存储                           │
│     • 便于实时播放（逐个时刻处理）                                 │
│     • 是最常用的存储格式                                           │
│                                                                      │
│  2. 平坦格式 (Planar / Non-interleaved)                             │
│  ───────────────────────────────────────                            │
│                                                                      │
│     相同声道的数据聚集排列                                         │
│                                                                      │
│     ┌────┬────┬────┬────┬────┬────┬────┬────┬────────┐            │
│     │ L0 │ L1 │ L2 │ L3 │... │ R0 │ R1 │ R2 │ ...    │            │
│     └────┴────┴────┴────┴────┴────┴────┴────┴────────┘            │
│     │←──── 左声道 ────→│    │←──── 右声道 ────→│                  │
│                                                                      │
│     特点：                                                          │
│     • 每个声道的数据连续存储                                       │
│     • 便于单独处理某个声道                                         │
│     • 某些音频处理库使用此格式                                     │
│                                                                      │
│  示例（16bit 立体声，4 个采样点）：                                │
│  ─────────────────────────────────                                  │
│                                                                      │
│     交错格式（16 字节）：                                          │
│     ┌──────────────────────────────────────────────────────────┐   │
│     │ L0[2B] R0[2B] L1[2B] R1[2B] L2[2B] R2[2B] L3[2B] R3[2B] │   │
│     └──────────────────────────────────────────────────────────┘   │
│                                                                      │
│     平坦格式（16 字节）：                                          │
│     ┌──────────────────────────────────────────────────────────┐   │
│     │ L0[2B] L1[2B] L2[2B] L3[2B] R0[2B] R1[2B] R2[2B] R3[2B] │   │
│     └──────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.3 字节序

```
┌─────────────────────────────────────────────────────────────────────┐
│                    字节序（Endianness）                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  对于 16bit 或更高位深的采样值，需要考虑字节序                     │
│                                                                      │
│  示例：采样值 0x1234 的存储方式                                    │
│                                                                      │
│  小端序 (Little-Endian)                                             │
│  ─────────────────────                                              │
│     ┌────────┬────────┐                                            │
│     │  0x34  │  0x12  │  低字节在前                                │
│     │  低位  │  高位  │                                            │
│     └────────┴────────┘                                            │
│     地址:  低    高                                                 │
│                                                                      │
│     • x86/x64、ARM（通常）使用小端序                               │
│     • 大多数 WAV 文件使用小端序                                    │
│                                                                      │
│  大端序 (Big-Endian)                                                │
│  ──────────────────                                                 │
│     ┌────────┬────────┐                                            │
│     │  0x12  │  0x34  │  高字节在前                                │
│     │  高位  │  低位  │                                            │
│     └────────┴────────┘                                            │
│     地址:  低    高                                                 │
│                                                                      │
│     • AIFF 文件使用大端序                                          │
│     • 网络传输通常使用大端序                                       │
│                                                                      │
│  处理 PCM 数据时需要注意：                                         │
│  ────────────────────────                                           │
│  • 读取/写入时需确认字节序                                         │
│  • 不同平台间传输可能需要转换                                      │
│  • Android/iOS 设备通常使用小端序                                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 7. AAC 编码详解

### 7.1 AAC 背景介绍

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AAC 背景介绍                                      │
├─────────────────────────────────────────────────────────────────────┤

│                                                                      │

│    - 在 MPEG-2 AAC 基础上增加：                                    │
│      · LTP（长时预测）                                             │
│      · PNS（知觉噪声替换）                                         │
│      · SBR（频段复制）                                             │
│      · PS（参数立体声）                                            │
│                                                                      │
│  AAC 与 MP3 对比：                                                  │
│  ─────────────────                                                  │
│                                                                      │
│  ┌────────────────┬──────────────────┬──────────────────┐          │
│  │     特性       │      MP3         │      AAC         │          │
│  ├────────────────┼──────────────────┼──────────────────┤          │
│  │ 发布年份       │ 1993             │ 1997/2000        │          │
│  │ 采样率支持     │ 16-48 kHz        │ 8-96 kHz         │          │
│  │ 声道数支持     │ 最多 2 声道      │ 最多 48 声道     │          │
│  │ 相同码率音质   │ 基准             │ 更好             │          │
│  │ 专利状态       │ 已过期（免费）   │ 有专利费         │          │
│  │ 应用场景       │ 音乐下载         │ 流媒体、直播     │          │
│  └────────────────┴──────────────────┴──────────────────┘          │
│                                                                      │
│  AAC 优势：                                                         │
│  ──────────                                                         │
│  • 相同比特率下，音质优于 MP3                                      │
│  • 支持更广泛的采样率（8kHz - 96kHz）                              │
│  • 支持多声道配置                                                  │
│  • 更高效的编码算法                                                │
│  • 广泛用于：iTunes、YouTube、短视频、直播                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 7.2 AAC 编码工具模块

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AAC 编码工具模块概览                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  AAC 属于感知音频编码（Perceptual Audio Coding）                   │
│  核心思想：利用人耳听觉掩蔽效应，去除不可感知的信息                │
│                                                                      │
│  主要编码模块：                                                     │
│  ─────────────                                                      │
│                                                                      │

│  │     • 将时域信号转换为频域                                  │   │
│  │     • 支持 128 或 1024 个频谱系数                          │   │
│  │     • 可切换长/短窗口适应信号特性                          │   │
│  │                                                              │   │
│  │  2. 心理声学模型 (Psychoacoustic Model)                     │   │
│  │     • 计算掩蔽阈值                                          │   │
│  │     • 分析频谱掩蔽效应                                      │   │
│  │     • 分析时域掩蔽效应                                      │   │
│  │     • 指导比特分配                                          │   │
│  │                                                              │   │
│  │  3. TNS (Temporal Noise Shaping) 瞬时噪声整形               │   │
│  │     • 控制量化噪声的时域分布                                │   │
│  │     • 对语音和剧烈变化信号效果显著                          │   │
│  │     • 在频域进行预测来修整时域噪声                          │   │
│  │                                                              │   │
│  │  4. 立体声编码                                               │   │
│  │     • M/S 立体声：合并相似的左右声道信息                   │   │
│  │     • Intensity 立体声：低频合并，高频用参数描述           │   │
│  │                                                              │   │
│  │  5. 预测 (Prediction)                                        │   │
│  │     • 利用帧间相关性减少冗余                                │   │
│  │     • 二阶后向自适应预测器                                  │   │
│  │                                                              │   │
│  │  6. 量化 (Quantization)                                      │   │
│  │     • 非均匀量化器                                          │   │
│  │     • 使用尺度因子控制噪声分布                              │   │
│  │     • 两层嵌套循环进行比特分配                              │   │
│  │                                                              │   │
│  │  7. 熵编码 (Entropy Coding)                                  │   │
│  │     • 改进的哈夫曼编码                                      │   │
│  │     • 无损压缩量化后的系数                                  │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

#### 9.1.1 AudioRecord 配置示例

```kotlin
// AudioRecord 配置示例
class AudioRecorder {
    
    private var audioRecord: AudioRecord? = null
    private var bufferSize: Int = 0
    
    // 录音参数
    companion object {
        const val SAMPLE_RATE = 44100          // 采样率
        const val CHANNEL_CONFIG = AudioFormat.CHANNEL_IN_STEREO  // 立体声
        const val AUDIO_FORMAT = AudioFormat.ENCODING_PCM_16BIT   // 16bit
    }
    
    fun init() {
        // 计算缓冲区大小
        bufferSize = AudioRecord.getMinBufferSize(
            SAMPLE_RATE,
            CHANNEL_CONFIG,
            AUDIO_FORMAT
        )
        
        // 创建 AudioRecord
        audioRecord = AudioRecord(
            MediaRecorder.AudioSource.MIC,  // 音频源：麦克风
            SAMPLE_RATE,                    // 采样率
            CHANNEL_CONFIG,                 // 声道配置
            AUDIO_FORMAT,                   // 音频格式
            bufferSize                      // 缓冲区大小
        )
    }
    
    fun startRecording(onData: (ByteArray, Int) -> Unit) {
        audioRecord?.startRecording()
        
        val buffer = ByteArray(bufferSize)
        while (isRecording) {
            val readSize = audioRecord?.read(buffer, 0, bufferSize) ?: 0
            if (readSize > 0) {
                onData(buffer, readSize)  // 回调 PCM 数据
            }
        }
    }
    
    fun stopRecording() {
        audioRecord?.stop()
        audioRecord?.release()
    }
}
```

#### 9.2.1 AAC 编码器配置示例

```kotlin
// MediaCodec AAC 编码器配置示例
class AACEncoder {
    
    private var mediaCodec: MediaCodec? = null
    
    companion object {
        const val SAMPLE_RATE = 44100
        const val CHANNEL_COUNT = 2
        const val BIT_RATE = 128000  // 128 kbps
    }
    
    fun configure() {
        // 创建 AAC 编码器
        mediaCodec = MediaCodec.createEncoderByType(MediaFormat.MIMETYPE_AUDIO_AAC)
        
        // 配置参数
        val format = MediaFormat.createAudioFormat(
            MediaFormat.MIMETYPE_AUDIO_AAC,
            SAMPLE_RATE,
            CHANNEL_COUNT
        ).apply {
            // 比特率
            setInteger(MediaFormat.KEY_BIT_RATE, BIT_RATE)
            
            // AAC Profile (LC = Low Complexity)
            setInteger(
                MediaFormat.KEY_AAC_PROFILE,
                MediaCodecInfo.CodecProfileLevel.AACObjectLC
            )
            
            // 最大输入大小
            setInteger(MediaFormat.KEY_MAX_INPUT_SIZE, 16384)
        }
        
        // 配置编码器
        mediaCodec?.configure(format, null, null, MediaCodec.CONFIGURE_FLAG_ENCODE)
    }
    
    fun start() {
        mediaCodec?.start()
    }
    
    fun encode(pcmData: ByteArray): ByteArray? {
        val codec = mediaCodec ?: return null
        
        // 获取输入缓冲区
        val inputBufferIndex = codec.dequeueInputBuffer(10000)
        if (inputBufferIndex >= 0) {
            val inputBuffer = codec.getInputBuffer(inputBufferIndex)
            inputBuffer?.clear()
            inputBuffer?.put(pcmData)
            codec.queueInputBuffer(
                inputBufferIndex,
                0,
                pcmData.size,
                System.nanoTime() / 1000,
                0
            )
        }
        
        // 获取输出缓冲区
        val bufferInfo = MediaCodec.BufferInfo()
        val outputBufferIndex = codec.dequeueOutputBuffer(bufferInfo, 10000)
        if (outputBufferIndex >= 0) {
            val outputBuffer = codec.getOutputBuffer(outputBufferIndex)
            val aacData = ByteArray(bufferInfo.size)
            outputBuffer?.get(aacData)
            codec.releaseOutputBuffer(outputBufferIndex, false)
            return aacData
        }
        
        return null
    }
    
    fun release() {
        mediaCodec?.stop()
        mediaCodec?.release()
    }
}
```

## 10. 总结

### 10.1 核心知识点回顾

```
┌─────────────────────────────────────────────────────────────────────┐
│                    音频编码核心知识点                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 声音基础                                                        │
│  ────────────                                                       │
│     • 声音 = 压力波（物体振动引起空气振动）                        │
│     • 三要素：频率（音高）、振幅（响度）、波形（音色）             │
│     • 人耳听觉范围：20Hz ~ 20kHz                                   │
│                                                                      │
│  2. 音频数字化                                                      │
│  ────────────────                                                   │
│     • 采样：时间轴离散化（44.1kHz）                                │
│     • 量化：幅度轴离散化（16bit = 65536级）                        │
│     • 编码：二进制存储（PCM）                                      │
│     • CD音质 = 44.1kHz × 16bit × 2ch ≈ 1411kbps                   │
│                                                                      │
│  3. 压缩原理                                                        │
│  ────────────                                                       │
│     • 时域冗余：样值相关性、静止系数                               │
│     • 频域冗余：功率谱非均匀性                                     │
│     • 听觉冗余：频谱掩蔽、时域掩蔽                                 │
│     • 有损压缩 vs 无损压缩                                         │
│                                                                      │
│  4. PCM 编码                                                        │
│  ────────────                                                       │
│     • 音频裸数据格式                                               │
│     • 交错格式 vs 平坦格式                                         │
│     • 大端序 vs 小端序                                             │
│                                                                      │
│  5. AAC 编码                                                        │
│  ────────────                                                       │
│     • 感知音频编码（利用掩蔽效应）                                 │
│     • 规格：LC / Main / SSR / LD / LTP / HE                        │
│     • 扩展技术：SBR（频段复制）、PS（参数立体声）                 │
│     • 广泛应用于流媒体、直播                                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```
