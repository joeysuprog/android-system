## 1. 概述与架构全景

### 1.1 RecyclerView 的设计目标

```
┌─────────────────────────────────────────────────────────────────────┐
│                RecyclerView 设计目标                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 高效的视图复用                                                  │
│     ─────────────────                                               │
│     • 四级缓存机制，最大化复用效率                                 │
│     • ViewHolder 模式强制使用                                      │
│     • 支持跨 RecyclerView 共享缓存池                              │
│                                                                      │
│  2. 灵活的布局方式                                                  │
│     ─────────────────                                               │
│     • LayoutManager 抽象布局逻辑                                   │
│     • 内置线性、网格、瀑布流布局                                  │
│     • 支持自定义布局管理器                                        │
│                                                                      │
│  3. 丰富的动画支持                                                  │
│     ─────────────────                                               │
│     • ItemAnimator 处理增删改动画                                 │
│     • 支持自定义动画效果                                          │
│     • 可预测的动画行为                                            │
│                                                                      │
│  4. 解耦的组件设计                                                  │
│     ─────────────────                                               │
│     • 职责分离：Adapter(数据) / LayoutManager(布局) / Animator(动画)│
│     • 各组件可独立替换                                            │
│     • 装饰器模式的 ItemDecoration                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 与 ListView 的对比

```
┌─────────────────────────────────────────────────────────────────────┐
│                ListView vs RecyclerView                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌──────────────────┬─────────────────┬─────────────────────────┐  │
│  │      特性        │    ListView     │     RecyclerView        │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  ViewHolder      │ 可选（需手动）   │ 强制使用                │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  缓存层级        │ 2 级            │ 4 级                    │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  布局方式        │ 仅垂直列表      │ 可自定义 LayoutManager │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  动画支持        │ 无内置          │ ItemAnimator            │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  分割线          │ divider 属性    │ ItemDecoration          │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  点击事件        │ OnItemClickListener │ 需自行实现         │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  局部刷新        │ 不支持          │ notifyItemXxx()         │  │
│  ├──────────────────┼─────────────────┼─────────────────────────┤  │
│  │  嵌套滑动        │ 不支持          │ NestedScrollingChild   │  │
│  └──────────────────┴─────────────────┴─────────────────────────┘  │
│                                                                      │
│  ListView 缓存机制（2 级）：                                       │
│  ─────────────────────────                                          │
│  • ActiveViews：当前屏幕可见的 View                                │
│  • ScrapViews：滑出屏幕的 View（按 ViewType 分类）                │
│                                                                      │
│  RecyclerView 缓存机制（4 级）：                                   │
│  ───────────────────────────                                        │
│  • Scrap：布局期间临时存储                                        │
│  • Cache：按 position 缓存，无需 rebind                           │
│  • ViewCacheExtension：自定义扩展                                 │
│  • RecycledViewPool：按 ViewType 缓存，需要 rebind               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 核心组件详解

### 2.1 Adapter 与 ViewHolder

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Adapter 与 ViewHolder                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ViewHolder 模式的意义：                                            │
│  ───────────────────────                                            │
│                                                                      │
│  传统方式（每次 getView 都 findViewById）：                        │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  fun getView(position: Int, convertView: View?): View {     │   │
│  │      val view = convertView ?: inflater.inflate(...)        │   │
│  │      val title = view.findViewById(R.id.title)  // 每次都查找│   │
│  │      val image = view.findViewById(R.id.image)  // 性能损耗 │   │
│  │      title.text = data[position].title                      │   │
│  │      return view                                             │   │
│  │  }                                                           │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ViewHolder 模式（缓存 View 引用）：                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  class MyViewHolder(view: View) : RecyclerView.ViewHolder(view) {│
│  │      val title: TextView = view.findViewById(R.id.title)    │   │
│  │      val image: ImageView = view.findViewById(R.id.image)   │   │
│  │      // View 引用被缓存，只在创建时查找一次                 │   │
│  │  }                                                           │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Adapter 核心方法：                                                 │
│  ─────────────────                                                  │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  abstract class Adapter<VH : ViewHolder> {                  │   │
│  │                                                              │   │
│  │      // 创建 ViewHolder（缓存未命中时调用）                 │   │
│  │      abstract fun onCreateViewHolder(                       │   │
│  │          parent: ViewGroup,                                 │   │
│  │          viewType: Int                                      │   │
│  │      ): VH                                                   │   │
│  │                                                              │   │
│  │      // 绑定数据到 ViewHolder                               │   │
│  │      abstract fun onBindViewHolder(holder: VH, position: Int)│   │
│  │                                                              │   │
│  │      // 带 payloads 的局部绑定（优化）                      │   │
│  │      open fun onBindViewHolder(                             │   │
│  │          holder: VH,                                        │   │
│  │          position: Int,                                     │   │
│  │          payloads: List<Any>                                │   │
│  │      )                                                       │   │
│  │                                                              │   │
│  │      // 数据数量                                            │   │
│  │      abstract fun getItemCount(): Int                       │   │
│  │                                                              │   │
│  │      // View 类型（多类型列表）                             │   │
│  │      open fun getItemViewType(position: Int): Int = 0       │   │
│  │                                                              │   │
│  │      // ViewHolder 回收时调用（清理资源）                   │   │
│  │      open fun onViewRecycled(holder: VH) {}                 │   │
│  │                                                              │   │
│  │      // ViewHolder 与 RecyclerView 关联/分离               │   │
│  │      open fun onViewAttachedToWindow(holder: VH) {}         │   │
│  │      open fun onViewDetachedFromWindow(holder: VH) {}       │   │
│  │  }                                                           │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  方法调用时机：                                                     │
│  ─────────────                                                      │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                                                               │  │
│  │  首次布局：                                                  │  │
│  │  ─────────                                                   │  │
│  │  position 0: onCreateViewHolder → onBindViewHolder          │  │
│  │  position 1: onCreateViewHolder → onBindViewHolder          │  │
│  │  position 2: onCreateViewHolder → onBindViewHolder          │  │
│  │  ...                                                         │  │
│  │                                                               │  │
│  │  滑动时（从 Cache 复用）：                                   │  │
│  │  ────────────────────────                                    │  │
│  │  position N: 无需任何调用，直接复用                         │  │
│  │                                                               │  │
│  │  滑动时（从 Pool 复用）：                                    │  │
│  │  ────────────────────────                                    │  │
│  │  position N: onBindViewHolder（需要重新绑定）               │  │
│  │                                                               │  │
│  │  回收时：                                                    │  │
│  │  ─────────                                                   │  │
│  │  ViewHolder 进入 Pool: onViewRecycled                       │  │
│  │                                                               │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 LayoutManager

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LayoutManager 详解                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  LayoutManager 的职责：                                             │
│  ─────────────────────                                              │
│  • 决定每个 item 的位置和大小                                      │
│  • 处理滑动逻辑                                                    │
│  • 决定何时回收不可见的 item                                       │
│  • 决定何时获取新的 item                                           │
│                                                                      │
│  内置 LayoutManager：                                               │
│  ─────────────────────                                              │
│                                                                      │
│  1. LinearLayoutManager（线性布局）                                │
│     ┌─────────────────────────────────────────┐                    │
│     │  ┌─────────────────────────────────┐   │                    │
│     │  │         Item 0                  │   │                    │
│     │  └─────────────────────────────────┘   │                    │
│     │  ┌─────────────────────────────────┐   │                    │
│     │  │         Item 1                  │   │                    │
│     │  └─────────────────────────────────┘   │                    │
│     │  ┌─────────────────────────────────┐   │                    │
│     │  │         Item 2                  │   │                    │
│     │  └─────────────────────────────────┘   │                    │
│     │              ...                       │                    │
│     └─────────────────────────────────────────┘                    │
│     支持垂直/水平方向，支持反向布局                               │
│                                                                      │
│  2. GridLayoutManager（网格布局）                                  │
│     ┌─────────────────────────────────────────┐                    │
│     │  ┌──────────┐ ┌──────────┐ ┌──────────┐│                    │
│     │  │  Item 0  │ │  Item 1  │ │  Item 2  ││                    │
│     │  └──────────┘ └──────────┘ └──────────┘│                    │
│     │  ┌──────────┐ ┌──────────┐ ┌──────────┐│                    │
│     │  │  Item 3  │ │  Item 4  │ │  Item 5  ││                    │
│     │  └──────────┘ └──────────┘ └──────────┘│                    │
│     └─────────────────────────────────────────┘                    │
│     继承自 LinearLayoutManager，支持 spanCount 和 spanSizeLookup  │
│                                                                      │
│  3. StaggeredGridLayoutManager（瀑布流布局）                       │
│     ┌─────────────────────────────────────────┐                    │
│     │  ┌──────────┐ ┌──────────┐ ┌──────────┐│                    │
│     │  │  Item 0  │ │          │ │  Item 2  ││                    │
│     │  │          │ │  Item 1  │ └──────────┘│                    │
│     │  └──────────┘ │          │ ┌──────────┐│                    │
│     │  ┌──────────┐ └──────────┘ │  Item 4  ││                    │
│     │  │  Item 3  │ ┌──────────┐ │          ││                    │
│     │  └──────────┘ │  Item 5  │ └──────────┘│                    │
│     └─────────────────────────────────────────┘                    │
│     每个 item 高度可不同，自动填充空隙                            │
│                                                                      │
│  自定义 LayoutManager 关键方法：                                   │
│  ─────────────────────────────                                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  abstract class LayoutManager {                              │   │
│  │                                                              │   │
│  │      // 生成默认 LayoutParams                               │   │
│  │      abstract fun generateDefaultLayoutParams(): LayoutParams│   │
│  │                                                              │   │
│  │      // 核心：布局所有 item                                 │   │
│  │      open fun onLayoutChildren(                             │   │
│  │          recycler: Recycler,                                │   │
│  │          state: State                                       │   │
│  │      ) {}                                                    │   │
│  │                                                              │   │
│  │      // 是否支持滑动                                        │   │
│  │      open fun canScrollHorizontally(): Boolean = false      │   │
│  │      open fun canScrollVertically(): Boolean = false        │   │
│  │                                                              │   │
│  │      // 处理滑动                                            │   │
│  │      open fun scrollHorizontallyBy(dx: Int, ...): Int = 0   │   │
│  │      open fun scrollVerticallyBy(dy: Int, ...): Int = 0     │   │
│  │                                                              │   │
```

### 3.2 Scrap 缓存详解

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Scrap 缓存                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Scrap 是布局期间的临时缓存，用于存储当前屏幕上的 ViewHolder       │
│                                                                      │
│  两种 Scrap：                                                       │
│  ───────────                                                        │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │

│  │  • 布局完成后清空                                          │   │
│  │                                                              │   │
│  │  mChangedScrap（变化废弃列表）                              │   │
│  │  ─────────────────────────────                              │   │
│  │  • 存储数据发生变化的 ViewHolder                           │   │
│  │  • 用于预布局（preLayout）计算动画                         │   │
│  │  • 仅在 mState.isPreLayout() 时使用                        │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Scrap 使用场景（布局期间）：                                      │
│  ───────────────────────────                                        │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  onLayoutChildren() {                                       │   │
│  │      // 1. 将所有 child 临时放入 Scrap                     │   │
│  │      detachAndScrapAttachedViews(recycler)                  │   │
│  │                                                              │   │
│  │      // 2. 填充 item（从 Scrap 中优先获取）                │   │
│  │      fill(recycler, layoutState, state)                     │   │
│  │                                                              │   │
│  │      // 3. 布局完成后，Scrap 中剩余的会被回收到 Pool       │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  示意图：                                                           │
│  ─────────                                                          │
│                                                                      │

│  │  ┌─────┐  │          │           │          │  ┌─────┐  │       │
│  │  │  A  │  │    →     │  Scrap:   │    →     │  │  A  │  │       │
│  │  ├─────┤  │          │  [A,B,C]  │          │  ├─────┤  │       │
│  │  │  B  │  │          │           │          │  │  B  │  │       │
│  │  ├─────┤  │          └───────────┘          │  ├─────┤  │       │
│  │  │  C  │  │                                 │  │  C  │  │       │
│  │  └─────┘  │                                 │  └─────┘  │       │
│  └───────────┘                                 └───────────┘       │

```

```
│  ─────────                                                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │

│  │  ──────────                                                 │   │
│  │                                                              │   │
│  │  ┌───────────────┐      ┌───────────────┐                   │   │
│  │  │   ┌─────┐     │      │               │                   │   │
│  │  │   │  0  │ ←───│──────│─── 滑出屏幕   │                   │   │
│  │  │   ├─────┤     │      │   ┌─────┐     │                   │   │
│  │  │   │  1  │     │  →   │   │  1  │     │                   │   │
│  │  │   ├─────┤     │      │   ├─────┤     │                   │   │
│  │  │   │  2  │     │      │   │  2  │     │                   │   │
│  │  │   ├─────┤     │      │   ├─────┤     │                   │   │
│  │  │   │  3  │     │      │   │  3  │     │                   │   │
│  │  │   └─────┘     │      │   ├─────┤     │                   │   │
│  │  └───────────────┘      │   │  4  │ ←───│─── 新显示         │   │
│  │                          │   └─────┘     │                   │   │
│  │                          └───────────────┘                   │   │
│  │                                                              │   │
│  │  item 0 滑出屏幕 → 进入 mCachedViews                        │   │
│  │                                                              │   │
│  │  mCachedViews: [position=0]                                 │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  继续向上滑动                                               │   │
│  │  ────────────                                               │   │
│  │                                                              │   │
│  │  item 1 滑出屏幕 → 进入 mCachedViews                        │   │
│  │                                                              │   │
│  │  mCachedViews: [position=0, position=1]  // 容量已满        │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  再继续向上滑动                                             │   │
│  │  ──────────────                                             │   │
│  │                                                              │   │
│  │  item 2 滑出屏幕 → 准备进入 mCachedViews                    │   │
│  │                                                              │   │
│  │  但 mCachedViews 已满！                                     │   │
│  │  → item 0 (最早进入的) 被移到 RecyclerPool                 │   │
│  │  → item 2 进入 mCachedViews                                │   │
│  │                                                              │   │
│  │  mCachedViews: [position=1, position=2]                     │   │
│  │  RecyclerPool: [viewType=0: 1个]                            │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  向下滑动回去                                               │   │
│  │  ────────────                                               │   │
│  │                                                              │   │
│  │  需要显示 item 2                                            │   │
│  │  → 从 mCachedViews 找到 position=2 → 命中！                │   │
│  │  → 直接复用，无需 rebind                                   │   │
│  │                                                              │   │
│  │  需要显示 item 1                                            │   │
│  │  → 从 mCachedViews 找到 position=1 → 命中！                │   │
│  │  → 直接复用，无需 rebind                                   │   │
│  │                                                              │   │
│  │  需要显示 item 0                                            │   │
│  │  → mCachedViews 找不到 position=0                          │   │
│  │  → 从 RecyclerPool 获取 → 命中！                           │   │
│  │  → 需要调用 onBindViewHolder() 重新绑定                    │   │
```

## 4. 布局流程

### 4.1 dispatchLayout 三步骤

```
┌─────────────────────────────────────────────────────────────────────┐
│                    dispatchLayout 布局流程                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  RecyclerView 的布局分三个步骤执行：                               │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  onLayout()                                                  │   │
│  │      │                                                       │   │
│  │      └─→ dispatchLayout()                                   │   │
│  │              │                                               │   │
│  │              ├─→ dispatchLayoutStep1()  // 预布局           │   │
│  │              │                                               │   │
│  │              ├─→ dispatchLayoutStep2()  // 实际布局         │   │
│  │              │                                               │   │
│  │              └─→ dispatchLayoutStep3()  // 后布局           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Step 1: dispatchLayoutStep1() - 预布局                            │
│  ─────────────────────────────────────────                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  作用：                                                      │   │
│  │  • 处理 Adapter 更新                                        │   │
│  │  • 记录动画前的 View 状态（preLayout）                      │   │
│  │  • 保存旧的 ViewHolder 信息用于后续动画计算                │   │
│  │                                                              │   │
│  │  void dispatchLayoutStep1() {                               │   │
│  │      // 处理待执行的 adapter 更新                          │   │
│  │      processAdapterUpdatesAndSetAnimationFlags();           │   │
│  │                                                              │   │
│  │      // 保存动画前的 View 信息                             │   │
│  │      mViewInfoStore.clear();                                │   │
│  │      for (child in children) {                              │   │
│  │          // 记录每个 View 的位置等信息                     │   │
│  │          mViewInfoStore.addToPreLayout(holder, info);       │   │
│  │      }                                                       │   │
│  │                                                              │   │
│  │      if (mState.mRunPredictiveAnimations) {                 │   │
│  │          // 预测性布局：为了动画                           │   │
│  │          mState.mInPreLayout = true;                        │   │
│  │          mLayout.onLayoutChildren(mRecycler, mState);       │   │
│  │      }                                                       │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Step 2: dispatchLayoutStep2() - 实际布局                          │
│  ─────────────────────────────────────────                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  作用：                                                      │   │
│  │  • 实际的布局过程                                          │   │
│  │  • 调用 LayoutManager.onLayoutChildren()                   │   │
│  │  • 确定所有 View 的最终位置                                │   │
│  │                                                              │   │
│  │  void dispatchLayoutStep2() {                               │   │
│  │      mState.mInPreLayout = false;                           │   │
│  │                                                              │   │
│  │      // 核心：调用 LayoutManager 进行布局                  │   │
│  │      mLayout.onLayoutChildren(mRecycler, mState);           │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Step 3: dispatchLayoutStep3() - 后布局                            │
│  ─────────────────────────────────────────                          │

│  │                                                              │   │
│  │  作用：                                                      │   │
│  │  • 记录动画后的 View 状态（postLayout）                    │   │
│  │  • 比较前后状态，触发相应动画                              │   │
│  │  • 清理临时数据                                            │   │
│  │                                                              │   │
│  │  void dispatchLayoutStep3() {                               │   │
│  │      // 记录布局后的 View 信息                             │   │
│  │      for (child in children) {                              │   │
│  │          mViewInfoStore.addToPostLayout(holder, info);      │   │
│  │      }                                                       │   │
│  │                                                              │   │
│  │      // 比较前后状态，触发动画                             │   │
│  │      mViewInfoStore.process(mViewInfoProcessCallback);      │   │
│  │                                                              │   │
│  │      // 清理                                                │   │
│  │      mLayout.removeAndRecycleScrapInt(mRecycler);           │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 LinearLayoutManager.onLayoutChildren()

```
┌─────────────────────────────────────────────────────────────────────┐
│              LinearLayoutManager.onLayoutChildren()                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  onLayoutChildren() 是 LayoutManager 的核心方法：                  │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  void onLayoutChildren(Recycler recycler, State state) {    │   │
│  │      // 1. 确定布局方向和锚点                               │   │
│  │      updateAnchorInfoForLayout(state, anchorInfo);          │   │
│  │                                                              │   │
│  │      // 2. 将所有 View 临时放入 Scrap                       │   │
│  │      detachAndScrapAttachedViews(recycler);                 │   │
│  │                                                              │   │
│  │      // 3. 从锚点向两个方向填充                             │   │
│  │      if (mAnchorInfo.mLayoutFromEnd) {                      │   │
│  │          // 从下往上布局                                    │   │
│  │          fill(recycler, layoutState, state); // 向上填充   │   │
│  │          fill(recycler, layoutState, state); // 向下填充   │   │
│  │      } else {                                                │   │
│  │          // 从上往下布局                                    │   │
│  │          fill(recycler, layoutState, state); // 向下填充   │   │
│  │          fill(recycler, layoutState, state); // 向上填充   │   │
│  │      }                                                       │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  锚点（Anchor）概念：                                              │
│  ─────────────────                                                  │
│                                                                      │
│  ┌─────────────────────────────────────────┐                       │
│  │                                          │                       │
│  │            ↑ 向上填充                    │                       │
│  │  ┌───────────────────────────────────┐  │                       │
│  │  │                                    │  │                       │
│  │  │                                    │  │                       │
│  │  ├────────────────────────────────────┤  │                       │
│  │  │  ★ Anchor Position (锚点)         │←─│─── 布局起点          │
│  │  ├────────────────────────────────────┤  │                       │
│  │  │                                    │  │                       │
│  │  │                                    │  │                       │
│  │  └───────────────────────────────────┘  │                       │
│  │            ↓ 向下填充                    │                       │
│  │                                          │                       │
│  └─────────────────────────────────────────┘                       │
│                                                                      │
│  fill() 填充方法：                                                  │
│  ─────────────────                                                  │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  int fill(Recycler recycler, LayoutState layoutState, ...) {│   │
│  │      int remainingSpace = layoutState.mAvailable;           │   │
│  │                                                              │   │
│  │      while (remainingSpace > 0 && layoutState.hasMore()) {  │   │
│  │          // 布局单个 item                                   │   │
│  │          layoutChunk(recycler, state, layoutState, result); │   │
│  │                                                              │   │
│  │          // 更新剩余空间                                    │   │
│  │          remainingSpace -= result.mConsumed;                │   │
│  │                                                              │   │
│  │          // 更新下一个位置                                  │   │
│  │          layoutState.mCurrentPosition += layoutState.mItemDirection;│
│  │      }                                                       │   │
│  │                                                              │   │
│  │      return start - remainingSpace; // 返回消耗的空间      │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  layoutChunk() 布局单个 item：                                     │
│  ─────────────────────────────                                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │

│  │      // 1. 获取 ViewHolder（核心：触发缓存查找）           │   │
│  │      View view = layoutState.next(recycler);                │   │
│  │      // 内部调用 recycler.getViewForPosition(position)     │   │
│  │                                                              │   │
│  │      // 2. 添加 View 到 RecyclerView                       │   │

│  │                                                              │   │
│  │      // 3. 测量 View                                        │   │
│  │      measureChildWithMargins(view, 0, 0);                   │   │
│  │                                                              │   │
│  │      // 4. 布局 View（确定位置）                           │   │
│  │      layoutDecoratedWithMargins(view, left, top, right, bottom);│
│  │  }                                                           │   │

```

```
│  ──────────────────────                                             │
│  • 由 ViewConfiguration.getScaledTouchSlop() 获取                  │
│  • 防止误触发滑动（通常 8dp）                                     │
│                                                                      │
│  onInterceptTouchEvent 关键逻辑：                                  │
│  ─────────────────────────────────                                  │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  boolean onInterceptTouchEvent(MotionEvent e) {             │   │
│  │      switch (action) {                                       │   │
│  │          case ACTION_DOWN:                                   │   │
│  │              // 记录初始触摸位置                            │   │
│  │              mInitialTouchX = (int) e.getX();               │   │
│  │              mInitialTouchY = (int) e.getY();               │   │
│  │              break;                                          │   │
│  │                                                              │   │
│  │          case ACTION_MOVE:                                   │   │
│  │              int dx = Math.abs(x - mInitialTouchX);         │   │
│  │              int dy = Math.abs(y - mInitialTouchY);         │   │
│  │                                                              │   │
│  │              // 判断是否应该拦截                            │   │
│  │              if (canScrollHorizontally && dx > mTouchSlop) {│   │
│  │                  mScrollState = SCROLL_STATE_DRAGGING;      │   │
│  │                  return true;  // 拦截                      │   │
│  │              }                                               │   │
│  │              if (canScrollVertically && dy > mTouchSlop) {  │   │
│  │                  mScrollState = SCROLL_STATE_DRAGGING;      │   │
│  │                  return true;  // 拦截                      │   │
│  │              }                                               │   │
│  │              break;                                          │   │
│  │      }                                                       │   │
│  │      return false;                                           │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.2 scrollByInternal 滑动实现

```
│                    scrollByInternal 滑动实现                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  滑动的核心流程：                                                   │
│  ───────────────                                                    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  scrollByInternal(dx, dy)                                   │   │
│  │      │                                                       │   │
│  │      ├─ 1. 触发嵌套滑动 dispatchNestedPreScroll()          │   │

│  │      │         mLayout.scrollVerticallyBy(dy, ...)         │   │
│  │      │                                                       │   │
│  │      ├─ 3. 触发嵌套滑动 dispatchNestedScroll()             │   │
│  │      │      → 通知父 View 剩余的滑动量                     │   │
│  │      │                                                       │   │

│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  LinearLayoutManager.scrollVerticallyBy()：                        │
│  ──────────────────────────────────────────                         │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  int scrollVerticallyBy(int dy, Recycler recycler, ...) {   │   │
│  │      // 委托给 scrollBy                                     │   │
│  │      return scrollBy(dy, recycler, state);                  │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  │  int scrollBy(int delta, Recycler recycler, State state) {  │   │
│  │      // 1. 更新布局状态                                     │   │
│  │      updateLayoutState(layoutDirection, requiredSpace);     │   │
│  │                                                              │   │
│  │      // 2. 填充新的 item（触发缓存机制）                   │   │
│  │      int scrolled = fill(recycler, mLayoutState, state);    │   │
│  │                                                              │   │
│  │      // 3. 回收滑出屏幕的 item                              │   │
│  │      recycleByLayoutState(recycler, mLayoutState);          │   │
│  │                                                              │   │
│  │      // 4. 偏移所有 child                                   │   │
│  │      mOrientationHelper.offsetChildren(-scrolled);          │   │

│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  滑动过程示意：                                                     │
│  ─────────────                                                      │
│                                                                      │
│  向上滑动 (dy > 0)                                                 │
│  ┌───────────────┐      ┌───────────────┐                         │
│  │   ┌─────┐     │      │               │                         │
│  │   │  0  │←────│──────│─── 回收       │                         │
│  │   ├─────┤     │      │   ┌─────┐     │                         │
│  │   │  1  │     │  →   │   │  1  │     │                         │
│  │   ├─────┤     │      │   ├─────┤     │                         │
│  │   │  2  │     │      │   │  2  │     │                         │
│  │   ├─────┤     │      │   ├─────┤     │                         │
│  │   │  3  │     │      │   │  3  │     │                         │
│  │   └─────┘     │      │   ├─────┤     │                         │
│  └───────────────┘      │   │  4  │←────│─── 填充（从缓存获取）   │
│                          │   └─────┘     │                         │
│                          └───────────────┘                         │
```

### 5.3 Fling 惯性滑动

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Fling 惯性滑动                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  当用户松开手指时，如果有足够的速度，会触发 Fling：               │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  ACTION_UP 时的处理                                         │   │
│  │      │                                                       │   │
│  │      ├─ 计算速度                                            │   │
│  │      │   mVelocityTracker.computeCurrentVelocity(1000);     │   │
│  │      │   float vx = mVelocityTracker.getXVelocity();        │   │
│  │      │   float vy = mVelocityTracker.getYVelocity();        │   │
│  │      │                                                       │   │
│  │      └─ 速度超过阈值?                                       │   │
│  │          ├─ Yes: fling(vx, vy)                              │   │
│  │          └─ No: 停止滑动                                    │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  fling() 实现：                                                     │
│  ─────────────                                                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  boolean fling(int velocityX, int velocityY) {              │   │
│  │      // 检查速度阈值                                        │   │
│  │      if (Math.abs(velocityX) < mMinFlingVelocity) {         │   │
│  │          velocityX = 0;                                      │   │
│  │      }                                                       │   │
│  │      if (Math.abs(velocityY) < mMinFlingVelocity) {         │   │
│  │          velocityY = 0;                                      │   │
│  │      }                                                       │   │
│  │                                                              │   │
│  │      // 启动 Fling                                          │   │
│  │      mViewFlinger.fling(velocityX, velocityY);              │   │
│  │      return true;                                            │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ViewFlinger（内部类）：                                           │
│  ──────────────────────                                             │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  class ViewFlinger implements Runnable {                    │   │
│  │      OverScroller mOverScroller;  // 滑动计算器            │   │
│  │                                                              │   │
│  │      void fling(int velocityX, int velocityY) {             │   │
│  │          mOverScroller.fling(                               │   │
│  │              0, 0,                    // 起始位置          │   │
│  │              velocityX, velocityY,    // 速度              │   │
│  │              MIN_VALUE, MAX_VALUE,    // X 范围            │   │
│  │              MIN_VALUE, MAX_VALUE     // Y 范围            │   │
│  │          );                                                  │   │
│  │          postOnAnimation();  // 开始动画循环               │   │
│  │      }                                                       │   │
│  │                                                              │   │
│  │      @Override                                               │   │
│  │      public void run() {                                    │   │
│  │          if (mOverScroller.computeScrollOffset()) {         │   │
│  │              // 计算本帧滑动量                              │   │
│  │              int dx = x - mLastX;                           │   │
│  │              int dy = y - mLastY;                           │   │
│  │                                                              │   │
│  │              // 执行滑动                                    │   │
│  │              scrollByInternal(dx, dy);                      │   │
│  │                                                              │   │
│  │              // 继续下一帧                                  │   │
│  │              postOnAnimation();                             │   │
│  │          }                                                   │   │
│  │      }                                                       │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Fling 动画流程：                                                   │
│  ───────────────                                                    │
│                                                                      │
│  手指抬起 → fling(velocity)                                        │
│      │                                                              │
│      └─→ ViewFlinger.run() (每帧执行)                              │
│              │                                                      │
│              ├─ OverScroller.computeScrollOffset() 计算位置        │
│              ├─ scrollByInternal() 执行滑动                        │
│              └─ postOnAnimation() 请求下一帧                       │
│                      │                                              │
│                      └─→ Choreographer 调度下一帧                  │
│                              │                                      │
│                              └─→ ViewFlinger.run() ...              │
│                                                                      │
│  直到 computeScrollOffset() 返回 false（滑动结束）                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 6. 数据更新机制

### 6.1 全量更新 vs 局部更新

```
┌─────────────────────────────────────────────────────────────────────┐
│                    数据更新方式对比                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  全量更新：notifyDataSetChanged()                                  │
│  ──────────────────────────────────                                 │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  adapter.notifyDataSetChanged()                             │   │
│  │      │                                                       │   │
│  │      ├─ 标记所有 ViewHolder 为 FLAG_INVALID                │   │
│  │      ├─ 所有 ViewHolder 不能复用到 Cache                   │   │
│  │      ├─ 只能进入 RecyclerPool（需要 rebind）               │   │
│  │      ├─ 触发 requestLayout()                               │   │
│  │      └─ 没有动画效果                                       │   │
│  │                                                              │   │
│  │  代价：                                                      │   │
│  │  • 所有可见 item 都会重新 bind                             │   │
│  │  • 无法利用 Cache 缓存                                     │   │
│  │  • 无动画                                                   │   │
│  │  • 滚动位置可能丢失                                        │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  局部更新：notifyItemXxx()                                         │
│  ─────────────────────────────                                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  // 插入                                                    │   │
│  │  adapter.notifyItemInserted(position)                       │   │
│  │  adapter.notifyItemRangeInserted(position, count)           │   │
│  │                                                              │   │
│  │  // 删除                                                    │   │
│  │  adapter.notifyItemRemoved(position)                        │   │
│  │  adapter.notifyItemRangeRemoved(position, count)            │   │
│  │                                                              │   │
│  │  // 更新                                                    │   │
│  │  adapter.notifyItemChanged(position)                        │   │
│  │  adapter.notifyItemChanged(position, payload)  // 局部刷新 │   │
│  │  adapter.notifyItemRangeChanged(position, count)            │   │
│  │                                                              │   │
│  │  // 移动                                                    │   │
│  │  adapter.notifyItemMoved(fromPosition, toPosition)          │   │
│  │                                                              │   │
│  │  优势：                                                      │   │
│  │  • 只更新受影响的 item                                     │   │
│  │  • Cache 缓存仍然有效                                      │   │
│  │  • 自动触发相应动画                                        │   │
│  │  • 性能更好                                                 │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  payload 局部刷新：                                                 │
│  ─────────────────                                                  │
│                                                                      │
│  // 只更新 item 的某个部分（如点赞数）                             │
│  adapter.notifyItemChanged(position, PAYLOAD_LIKE_COUNT)           │
│                                                                      │
│  override fun onBindViewHolder(                                     │
│      holder: VH,                                                    │
│      position: Int,                                                 │
│      payloads: List<Any>                                           │
│  ) {                                                                │
│      if (payloads.isEmpty()) {                                      │
│          // 全量绑定                                               │
│          onBindViewHolder(holder, position)                        │
│      } else {                                                       │
│          // 局部绑定                                               │
│          for (payload in payloads) {                               │
│              when (payload) {                                       │
│                  PAYLOAD_LIKE_COUNT -> {                           │
│                      holder.likeCount.text = data[position].likes  │
│                  }                                                  │
│              }                                                      │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.2 DiffUtil

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DiffUtil 差异计算                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  DiffUtil 自动计算两个列表的差异，生成最优更新操作：              │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  Old List         DiffUtil.calculateDiff()       New List   │   │
│  │  ┌───────┐                │                     ┌───────┐   │   │
│  │  │   A   │                │                     │   A   │   │   │
│  │  │   B   │        ────────┼────────→           │   C   │   │   │
│  │  │   C   │                │                     │   B   │   │   │
│  │  │   D   │                ↓                     │   E   │   │   │
│  │  └───────┘                                      └───────┘   │   │
│  │                  DiffResult:                                │   │
│  │                  • B: moved from 1 to 2                     │   │
│  │                  • D: removed at 3                          │   │
│  │                  • E: inserted at 3                         │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  使用方式：                                                         │
│  ─────────                                                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  // 1. 实现 DiffUtil.Callback                               │   │
│  │  class MyDiffCallback(                                      │   │
│  │      private val oldList: List<Item>,                       │   │
│  │      private val newList: List<Item>                        │   │
│  │  ) : DiffUtil.Callback() {                                  │   │
│  │                                                              │   │
│  │      override fun getOldListSize() = oldList.size           │   │
│  │      override fun getNewListSize() = newList.size           │   │
│  │                                                              │   │
│  │      // 判断是否同一个 item（通常比较 id）                  │   │
│  │      override fun areItemsTheSame(oldPos: Int, newPos: Int): Boolean {│
│  │          return oldList[oldPos].id == newList[newPos].id    │   │
│  │      }                                                       │   │
│  │                                                              │   │
│  │      // 判断内容是否相同（用于决定是否需要更新）            │   │
│  │      override fun areContentsTheSame(oldPos: Int, newPos: Int): Boolean {│
│  │          return oldList[oldPos] == newList[newPos]          │   │
│  │      }                                                       │   │
│  │                                                              │   │
│  │      // 可选：返回变化的部分（用于 payload 局部刷新）       │   │
│  │      override fun getChangePayload(oldPos: Int, newPos: Int): Any? {│
│  │          val old = oldList[oldPos]                          │   │
│  │          val new = newList[newPos]                          │   │
│  │          if (old.likeCount != new.likeCount) {              │   │
│  │              return PAYLOAD_LIKE_COUNT                      │   │
│  │          }                                                   │   │
│  │          return null                                         │   │
│  │      }                                                       │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  │  // 2. 计算差异并应用                                       │   │
│  │  val diffResult = DiffUtil.calculateDiff(                   │   │
│  │      MyDiffCallback(oldList, newList)                       │   │
│  │  )                                                           │   │
│  │                                                              │   │
│  │  // 3. 应用到 Adapter                                       │   │
│  │  diffResult.dispatchUpdatesTo(adapter)                      │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  Myers Diff 算法：                                                  │
│  ─────────────────                                                  │
│  • 时间复杂度：O(N + D²)，D 是差异数                              │
│  • 空间复杂度：O(N)                                               │
│  • 对于大列表，建议在后台线程计算                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.3 AsyncListDiffer / ListAdapter

```

│                                                                      │

│  │      }                                                       │   │
│  │  }                                                           │   │
│  │                                                              │   │
│  │  // 提交新列表（自动在后台计算差异）                        │   │
│  │  adapter.submitList(newList)                                │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  工作流程：                                                         │
│  ─────────                                                          │
│                                                                      │
│  submitList(newList)                                               │
│      │                                                              │
│      │ 主线程                                                      │
│      ↓                                                              │
│  AsyncDifferConfig.getBackgroundExecutor()                         │
│      │                                                              │
│      │ 后台线程                                                    │
│      ↓                                                              │
│  DiffUtil.calculateDiff()  ← 耗时计算在后台                       │
│      │                                                              │
│      │ 计算完成                                                    │
│      ↓                                                              │
│  MainExecutor (主线程)                                             │
│      │                                                              │
│      └─→ diffResult.dispatchUpdatesTo(adapter)                     │
│                                                                      │
│  注意事项：                                                         │
│  ─────────                                                          │
│  • submitList 是异步的，新数据不会立即生效                        │
│  • 提交相同引用的列表不会触发更新（需要创建新列表）               │
│  • 可以传入回调监听更新完成：submitList(list) { onCommitted() }   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 8. 性能优化

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RecyclerView 性能优化                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. setHasFixedSize(true)                                          │
│     ────────────────────────                                        │
│     当 RecyclerView 大小不会因为内容变化而改变时使用              │
│                                                                      │
│     recyclerView.setHasFixedSize(true)                             │
│                                                                      │
│     效果：跳过 requestLayout() 中的 measure 步骤                  │
│                                                                      │
│  2. setItemViewCacheSize()                                         │
│     ──────────────────────────                                      │
│     增加 Cache 缓存大小                                            │
│                                                                      │
│     recyclerView.setItemViewCacheSize(10)  // 默认 2              │
│                                                                      │
│     适用：快速来回滑动的场景                                      │
│                                                                      │
│  3. RecycledViewPool 共享                                          │
│     ────────────────────────                                        │
│     多个 RecyclerView 共享缓存池                                  │
│                                                                      │
│     val pool = RecycledViewPool()                                  │
│     pool.setMaxRecycledViews(0, 10)  // 增加容量                  │
│     recyclerView1.setRecycledViewPool(pool)                        │
│     recyclerView2.setRecycledViewPool(pool)                        │
│                                                                      │
│     适用：ViewPager + 多个相同类型的 RecyclerView                 │
│                                                                      │
│  4. setHasStableIds(true)                                          │
│     ────────────────────────                                        │
│     启用稳定 ID，优化数据更新时的复用                             │
│                                                                      │
│     init { setHasStableIds(true) }                                 │
│     override fun getItemId(position: Int) = data[position].id     │
│                                                                      │
│     效果：数据更新时可以根据 id 复用 ViewHolder                   │
│                                                                      │
│  5. 避免在 onBindViewHolder 中创建对象                            │
│     ──────────────────────────────────────                          │
│     // 错误：每次都创建新对象                                     │
│     override fun onBindViewHolder(holder: VH, position: Int) {     │
│         holder.itemView.setOnClickListener { ... }  // 每次创建   │
│     }                                                               │
│                                                                      │
│     // 正确：在 ViewHolder 中保存或使用位置判断                   │
│     class VH(view: View) : RecyclerView.ViewHolder(view) {         │
│         init {                                                      │
│             view.setOnClickListener { onClick(bindingAdapterPosition) }│
│         }                                                           │
│     }                                                               │
│                                                                      │
│  6. 使用 payload 局部刷新                                          │
│     ─────────────────────────                                       │
│     // 只更新变化的部分                                            │
│     adapter.notifyItemChanged(position, PAYLOAD_PROGRESS)          │
│                                                                      │
│  7. 图片加载优化                                                   │
│     ─────────────                                                   │
│     • 使用合适的图片尺寸                                          │
│     • 滑动时暂停加载，停止后恢复                                  │
│     • 使用 placeholder 占位图                                      │
│                                                                      │
│     recyclerView.addOnScrollListener(object : OnScrollListener() { │
│         override fun onScrollStateChanged(rv: RV, state: Int) {    │
│             when (state) {                                          │
│                 SCROLL_STATE_IDLE -> Glide.resumeRequests(context) │
│                 else -> Glide.pauseRequests(context)               │
│             }                                                       │
│         }                                                           │
│     })                                                              │
│                                                                      │
│  8. 减少布局层级                                                   │
│     ─────────────                                                   │
│     • 使用 ConstraintLayout 扁平化布局                            │
│     • 避免过度嵌套                                                 │
│                                                                      │
│  9. 预加载                                                         │
│     ─────                                                           │
│     // 设置预取数量                                                │
│     (layoutManager as LinearLayoutManager)                         │
│         .initialPrefetchItemCount = 5                              │
│                                                                      │
│  10. 关闭动画（极端场景）                                         │
│      ────────────────────                                           │
│      recyclerView.itemAnimator = null                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```
