# Day 0：环境与工具准备

## 1. AOSP 源码拉取

### 使用清华镜像源（推荐国内用户）

```bash
# 设置 repo 工具镜像
export REPO_URL='https://mirrors.tuna.tsinghua.edu.cn/git/git-repo'

# 初始化仓库（Android 15）
repo init -u https://mirrors.tuna.tsinghua.edu.cn/git/AOSP/platform/manifest \
  -b android-15.0.0_r1 --depth=1

# 全量同步（约 200-300GB，建议挂在后台跑一晚上）
repo sync -c -j8 --no-tags --force-sync
```

> **说明**：拉取全量代码是为了整体了解 AOSP 架构，学习时重点精读下方"核心源码目录"即可。
> 全量同步预计耗时 4-12 小时（取决于网络），可以先用 `cs.android.com` 在线浏览开始学习。

### Android 15 相较于 14 的重要变化（学习时关注）

| 模块 | 变化 | 对学习的影响 |
|------|------|-------------|
| **SurfaceFlinger** | 引入 `Scheduler` 重构，更灵活的帧率管理 | 学习 VSync 调度时关注新的 Scheduler 代码 |
| **HWUI** | Vulkan 渲染管线进一步优化，部分设备默认使用 Vulkan | 关注 `SkiaVulkanPipeline` 相关代码 |
| **SystemUI** | Scene Framework 引入，模块化重构持续推进 | 阅读 SystemUI 时注意新的 Scene 架构 |
| **窗口管理** | Desktop Windowing（桌面窗口模式）增强 | WMS 中新增 freeform 窗口相关逻辑 |
| **隐私/安全** | 前台服务类型更严格、部分权限收紧 | AMS 中的权限检查逻辑更完善 |
| **性能** | 应用启动优化、ART 编译器改进 | Perfetto 中可观察到更详细的 ART trace |

## 2. 工具链验证

### adb 命令验证

```bash
# 查看版本
adb version

# 连接设备后验证
adb devices
adb shell getprop ro.build.version.release   # Android 版本
adb shell getprop ro.build.display.id         # 系统构建号

# 常用调试命令验证
adb shell dumpsys -l | wc -l                  # 系统服务数量
adb shell dumpsys SurfaceFlinger --list        # SurfaceFlinger Layer 列表
```

### Perfetto 工具准备

**方法 1：通过 adb 命令行抓取**

```bash
# 基础抓取（10 秒，覆盖渲染相关 trace 类别）
adb shell perfetto -o /data/misc/perfetto-traces/trace.pb -t 10s \
  sched freq idle am wm gfx view binder_driver hal dalvik camera input res memory

# 拉取到本地
adb pull /data/misc/perfetto-traces/trace.pb ./trace.pb
```

**方法 2：通过 Perfetto UI 在线录制**

1. 打开 https://ui.perfetto.dev
2. 点击 "Record new trace"
3. 选择已连接的 Android 设备
4. 勾选需要的 trace 类别（推荐：CPU scheduling、GPU rendering、Android framework）
5. 设置录制时长（10 秒即可）
6. 点击 "Start recording"

**方法 3：自定义 trace config（进阶）**

创建 `trace_config.pbtx` 文件：

```protobuf
buffers: {
    size_kb: 63488
    fill_policy: RING_BUFFER
}
data_sources: {
    config {
        name: "linux.ftrace"
        ftrace_config {
            ftrace_events: "sched/sched_switch"
            ftrace_events: "sched/sched_wakeup"
            ftrace_events: "power/suspend_resume"
            ftrace_events: "power/cpu_frequency"
            atrace_categories: "gfx"
            atrace_categories: "view"
            atrace_categories: "wm"
            atrace_categories: "am"
            atrace_categories: "input"
            atrace_categories: "sched"
            atrace_apps: "*"
        }
    }
}
data_sources: {
    config {
        name: "linux.process_stats"
        process_stats_config {
            scan_all_processes_on_start: true
        }
    }
}
duration_ms: 10000
```

```bash
adb push trace_config.pbtx /data/local/tmp/
adb shell perfetto -c /data/local/tmp/trace_config.pbtx \
  -o /data/misc/perfetto-traces/trace.pb
adb pull /data/misc/perfetto-traces/trace.pb
```

### 开发者选项配置（真机）

在设备上依次打开：

1. **设置 → 关于手机 → 连续点击"版本号" 7 次** → 开启开发者选项
2. **开发者选项** 中打开：
   - USB 调试
   - GPU 渲染模式分析（Profile GPU Rendering） → 选"在屏幕上显示为条形图"
   - 显示过度绘制区域（Debug GPU overdraw）
   - 窗口动画缩放 / 过渡动画缩放 / Animator 时长缩放 → 可调为 0.5x 方便观察

## 3. Cursor 中阅读源码技巧

- 使用 `Cmd+P` 快速打开文件（输入类名如 `ViewRootImpl`）
- 使用 `Cmd+Shift+F` 全局搜索关键方法名（如 `performTraversals`）
- 使用 `Cmd+Click` 跳转到定义
- 与 AI 对话时，直接 `@` 引用源码文件，让 AI 讲解关键方法
